# 贡献指南

感谢您对 WeMediaSpider 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告 Bug

如果您发现了 bug，请：

1. 在 [Issues](https://github.com/vag-Zhao/WeMediaSpider/issues) 中搜索是否已有相关报告
2. 如果没有，创建一个新的 Issue，使用 Bug 反馈模板
3. 提供详细的复现步骤和环境信息

### 提出新功能

如果您有好的想法：

1. 先在 Issues 中讨论，确保该功能符合项目方向
2. 使用功能请求模板创建 Issue
3. 等待维护者反馈

### 提交代码

1. Fork 本项目
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 打开一个 Pull Request

### 代码规范

- 遵循 PEP 8 Python 代码风格
- 添加必要的注释和文档字符串
- 确保代码可以通过基本测试
- 保持代码简洁易读

### 提交信息规范

使用清晰的提交信息：

- `feat:` 新功能
- `fix:` 修复 bug
- `docs:` 文档更新
- `style:` 代码格式调整
- `refactor:` 重构
- `test:` 测试相关
- `chore:` 构建/工具相关

示例：
```
feat: 添加文章导出为 PDF 功能
fix: 修复登录二维码刷新问题
docs: 更新安装说明
```

## 行为准则

- 尊重所有贡献者
- 建设性地提出问题和建议
- 接受建设性的批评
- 专注于对社区最有利的事情

## 许可

通过贡献代码，您同意您的贡献将按照 MIT 许可证进行授权。
